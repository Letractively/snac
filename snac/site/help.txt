<p>Теперь, когда SNAC установлен, вы можете приступить к&nbsp;настройке сайта.</p>
<h3>Настройка внешнего вида</h3>

<p>В&nbsp;директории <code>site/</code> вы&nbsp;найдете файлы <code>template.html</code> (шаблон страницы) и&nbsp;<code>styles.css</code> (таблица стилей). Эти два файла определяют внешний вид всех страниц сайта. Если вы&nbsp;знакомы с&nbsp;HTML и&nbsp;CSS, создание собственного дизайна сайта не&nbsp;должно вызвать сложностей. Если нет&nbsp;&mdash; попросите <nobr>кого-нибудь</nobr> сделать это за&nbsp;вас&nbsp;<nobr>:-)</nobr></p>
<p>Шаблон страницы должен состоять только из&nbsp;содержимого тега <code>body</code>. Те&nbsp;места в&nbsp;шаблоне, куда в&nbsp;дальнейшем будет помещаться динамическое содержимое (заголовок страницы, основной текст, боковая панель и&nbsp;прочее), следует разметить следующим образом:</p>
<p><code><$= this.field_name $></code></p>
<p>Здесь <code>field_name</code>&nbsp;&mdash; название поля, латинскими буквами и&nbsp;без пробелов. Например, простая страница с&nbsp;названием сайта, заголовком, текстом и&nbsp;копирайтом будет выглядеть так:</p>
<p><code>&lt;h1&gt;&lt;$= this.title $&gt;&lt;/h1&gt;<br />
&lt;h2&gt;&lt;$= this.header $&gt;&lt;/h2&gt;<br />
&lt;div class="content"&gt;&lt;$= this.content $&gt;&lt;/div&gt;<br />
&lt;div class="footer"&gt;&amp;copy; &lt;$= this.author $&gt;&lt;/div&gt;</code></p>
<blockquote><strong>Примечание.</strong> SNAC использует шаблонизатор jQote2, с&nbsp;его более продвинутыми возможностями по&nbsp;настройке шаблонов можно ознакомиться на&nbsp;<a href="http://aefxx.com/jquery-plugins/jqote2/">сайте автора</a>.</blockquote>
<p>Редактировать файлы страниц на&nbsp;сервере очень удобно с&nbsp;помощью <nobr>FTP-клиента</nobr> <a href="http://filezilla.ru/">FileZilla</a> и&nbsp;текстового редактора <a href="http://notepad-plus-plus.org/">Notepad++</a>:</p>
<p><img src="site/images/editing.png" alt="Редактирование страницы" /></p>
<p>Для корректного отображения кириллицы, сохраните шаблон в&nbsp;кодировке <nobr>UTF-8</nobr>.</p
<h3>Редактирование и&nbsp;добавление данных на&nbsp;страницу</h3>

<p>Данные каждой страницы хранятся в&nbsp;отдельном текстовом файле в&nbsp;директории <code>site/</code>. Название файла рекомендуется давать латинскими буквами без пробелов. Название файла является адресом страницы (например, данные страницы <code>www.example.com/#!about</code> хранятся в&nbsp;файле <code>site/about.txt</code>).</p>
<p>Данные страницы должны быть записаны в&nbsp;следующем формате:</p>
<p><code>$ field_name $<br />
value</code></p>
<p>Здесь <code>field_name</code>&nbsp;&mdash; название поля латинскими буквами без пробелов, а&nbsp;<code>value</code>&nbsp;&mdash; значение поля. Именно на&nbsp;это название поля следует ссылаться из&nbsp;шаблона страницы. Например, для написанного нами шаблона:</p>
<p><code>&lt;h2&gt;&lt;$= this.header $&gt;&lt;/h2&gt;<br />
&lt;div class="content"&gt;&lt;$= this.content $&gt;&lt;/div&gt;</code></p>
<p>файл страницы должен содержать следующие поля:</p>
<p><code>$ header $<br />
Контактная информация<br />
<br />$ content $<br />
mail@example.com</code></p>
<p>Никакие строки, кроме строк с&nbsp;названиями полей, не&nbsp;должны начинаться со&nbsp;знака <code>$</code>. Файл может содержать любое количество полей. Единственное обязательное поле&nbsp;&mdash; <code>header</code>, его значение будет помещено в&nbsp;заголовок окна. Отсутствие этого поля приведет к&nbsp;ошибке при попытке открыть страницу.</p>
<p>Для корректной работы с&nbsp;кириллицей, сохраняйте все файлы в&nbsp;кодировке <nobr>UTF-8</nobr>.</p>

<h3>Настройка общей информации о&nbsp;сайте</h3>

<p>Общая информация о&nbsp;сайте, то&nbsp;есть такие данные, которые не&nbsp;меняются от&nbsp;страницы к&nbsp;странице, хранятся в&nbsp;файле <code>global.txt</code>. Отсутствие этого файла приводит к&nbsp;ошибке.</p>
<p>Формат файла <code>global.txt</code> такой&nbsp;же, как и&nbsp;файлов с&nbsp;данными страниц. Единственное различие заключается в&nbsp;том, что поля из&nbsp;<code>global.txt</code> доступны на&nbsp;каждой странице. Пример файла <code>global.txt</code>:</p>
<p><code>$ title $<br />
Мой сайт<br />
<br />$ author $<br />
Василий Пупкин</code></p>
<h3>Ссылки и&nbsp;меню</h3>

<p>Чтобы дать ссылку на&nbsp;другую страницу, укажите ее&nbsp;имя после знаков <code>#!</code>: <code>&lt;a href="#page"&gt;ссылка&lt;/a&gt;</code></p><p>Меню&nbsp;&mdash; это перечисление ссылок на&nbsp;другие страницы в&nbsp;рамках одного блока. SNAC автоматически присвоит ссылке на&nbsp;текущую страницу класс <code><nobr>snac-current</nobr></code>. Чтобы эта функция заработала, в&nbsp;файле <code>global.txt</code> добавьте строку</p><p><code>$ menu $<br />
selector</code></p><p>где <code>selector</code>&nbsp;&mdash; это селектор блока, содержащего меню, в&nbsp;стиле jQuery (<a href="http://api.jquery.com/category/selectors/">подробнее про&nbsp;селекторы jQuery</a>). Например, если меню находится в&nbsp;списке с&nbsp;классом <code>my_menu</code>, то запись должна быть следующей:</p>
<p><code>$ menu $<br />
ul.my_menu</code></p>

<h3>Ошибка 404</h3>

<p>Если посетитель сайта переходит на&nbsp;несуществующею страницу, SNAC перенаправляет его на&nbsp;страницу <code>#!404</code>. Вы&nbsp;можете настроить ее&nbsp;содержание редактированием файла <code>site/404.txt</code>.</p>

<h3>Создание бэкапа</h3>

<p>Для полного восстановления сайта достаточно иметь резервную копию директории <code>site/</code>. В&nbsp;ней хранится как содержимое страниц, так и&nbsp;шаблон оформления. Директорию <code>system/</code> и&nbsp;файл <code>index.html</code> всегда можно скачать с&nbsp;этого сайта.</p>
