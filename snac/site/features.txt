<p>Основная задача SNAC&nbsp;&mdash; отделить оформление страниц сайта от&nbsp;их&nbsp;содержимого. Оформление всех страниц задается шаблоном, в&nbsp;котором отмечаются места, куда будет добавлено содержимое. Само содержимое хранится в&nbsp;отдельных текстовых файлах. SNAC помещает содержимое в&nbsp;шаблон и&nbsp;генерирует страницу, вот так:</p>
<p><img src="site/images/generation.png" alt="Как работает SNAC" /></p>

<h3>Какие еще есть фишки?</h3>

<p>Можно:</p>
<ul><li>Установить SNAC простым копированием</li><li>поменять оформление и&nbsp;разметку всех страниц сайта сразу,</li><li>использовать продвинутые возможности jQote для настройки шаблона,</li><li>хранить содержимое &laquo;глобально&raquo;, чтобы его можно было поместить на&nbsp;несколько страниц сразу,</li><li>автоматически выделять текущую страницу в&nbsp;меню,</li><li>установить SNAC даже на&nbsp;бесплатный хостинг без серверных технологий, потому что <strong>SNAC работает полностью на&nbsp;стороне клиента</strong>,</li><li>хранить локальную копию (бэкап) сайта и&nbsp;восстановить сайт простым копированием.</li></ul>
<p>Подробнее про&nbsp;использование SNAC рассказано в&nbsp;разделе &laquo;<a href="#!help">Как пользоваться</a>&raquo.
<h3>Что пока не&nbsp;сделано, но&nbsp;будет?</h3>

<p>Содержимое страниц пока оформляется только с&nbsp;помощью HTML, но&nbsp;будет добавлен Markdown.</p>
<p>Только один шаблон на&nbsp;весь сайт, невозможно разные страницы оформить <nobr>по-разному</nobr>. Возможно, будет добавлена поддержка нескольких шаблонов.</p>

<h3>Чего сделано никогда не&nbsp;будет?</h3>

<p>Поскольку SNAC работает на&nbsp;стороне клиента, он&nbsp;не&nbsp;может сделать некоторые вещь в&nbsp;принципе. Например:</p>
<ul><li>Нельзя создавать иерархию страниц.</li><li>Нельзя сделать полнотекстовый поиск по&nbsp;сайту.</li><li>Нельзя автоматически сгенерировать меню.</li></ul>
<p>К&nbsp;счастью, SNAC&nbsp;&mdash; не&nbsp;система управления сайтом, поэтому уметь делать эти вещи он&nbsp;не&nbsp;обязан.</p>
