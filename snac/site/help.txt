<h3>Адреса страниц</h3>

<p>При переходе по&nbsp;страницам тестового сайта SNAC вы&nbsp;могли заметить необычный вид адресов страниц: перед названием страницы стоят символы <code>#!</code>. Эта специальная последовательность, которая сигнализируют SNAC о&nbsp;необходимости совершить переход на&nbsp;другую страницу сайта. Чтобы дать ссылку на&nbsp;другую страницу, укажите ее&nbsp;имя после знаков <code>#!</code>:</p>
<p><code>&lt;a href="#!<em>page</em>"&gt;ссылка&lt;/a&gt;</code></p>
<h3>Настройка внешнего вида</h3>

<p>В&nbsp;директории <code>site/</code> вы&nbsp;найдете файлы <code>template.html</code> (шаблон страницы) и&nbsp;<code>styles.css</code> (таблица стилей). Эти два файла определяют внешний вид всех страниц сайта. Если вы&nbsp;знакомы с&nbsp;HTML и&nbsp;CSS, создание собственного дизайна сайта не&nbsp;должно вызвать сложностей. Если нет&nbsp;&mdash; попросите <nobr>кого-нибудь</nobr> сделать это за&nbsp;вас&nbsp;<nobr>:-)</nobr></p>
<p>Шаблон страницы должен состоять только из&nbsp;кода, который вы&nbsp;поместили&nbsp;бы внутрь тега <code>body</code>. В&nbsp;нем не&nbsp;должны упоминаться теги <code>html</code>, <code>head</code> и&nbsp;сам тег <code>body</code>.</p>
<p>Чтобы указать системе те&nbsp;места в&nbsp;шаблоне, куда следует помещать содержание страницы, отметьте их&nbsp;с&nbsp;помощью специального синтаксиса:</p>
<p><code><$= this.<em>field_name</em> $></code></p>
<p>Здесь <code>field_name</code>&nbsp;&mdash; название поля, латинскими буквами и&nbsp;без пробелов. Например, простая страница с&nbsp;названием сайта, заголовком, текстом и&nbsp;копирайтом будет выглядеть так:</p>
<p><code>&lt;h1&gt;&lt;$= this.title $&gt;&lt;/h1&gt;<br />
&lt;h2&gt;&lt;$= this.header $&gt;&lt;/h2&gt;<br />
&lt;div class="content"&gt;&lt;$= this.content $&gt;&lt;/div&gt;<br />
&lt;div class="footer"&gt;&amp;copy; &lt;$= this.author $&gt;&lt;/div&gt;</code></p>
<blockquote><strong>Примечание для тех, кто знаком с&nbsp;Javascript.</strong> SNAC использует шаблонизатор jQote2, его возможности гораздо шире простой подстановки значений. С&nbsp;более продвинутыми возможностями по&nbsp;настройке шаблонов можно ознакомиться на&nbsp;<a href="http://aefxx.com/jquery-plugins/jqote2/">сайте автора</a>.</blockquote>
<p>Редактировать файлы страниц можно с&nbsp;помощью любого удобного средства. Лично я использую <nobr>FTP-клиент</nobr> <a href="http://filezilla.ru/">FileZilla</a> и&nbsp;текстовый редактора <a href="http://notepad-plus-plus.org/">Notepad++</a> для редактирования файлов прямо на&nbsp;сервере:</p>
<p><img src="site/images/editing.png" alt="Редактирование страницы" /></p>
<p>Для корректного отображения кириллицы, сохраните шаблон в&nbsp;кодировке <nobr>UTF-8</nobr>.</p>
<h3>Редактирование и&nbsp;добавление данных на&nbsp;страницу</h3>

<p>Содержание страницы хранится в&nbsp;отдельном текстовом файле в&nbsp;директории <code>site/</code>. Название файла рекомендуется давать латинскими буквами без пробелов. Название файла является адресом страницы (например, данные страницы <code>www.example.com/#!about</code> хранятся в&nbsp;файле <code>site/about.txt</code>).</p>
<p>Содержание страницы в&nbsp;текстовом файле должно иметь особый формат:</p>
<p><code>$ <em>field_name</em> $<br />
<em>content</em></code></p>
<p>Здесь <code>field_name</code>&nbsp;&mdash; название поля латинскими буквами без пробелов, а&nbsp;<code>content</code>&nbsp;&mdash; содержание поля. Именно на&nbsp;это название поля следует ссылаться из&nbsp;шаблона страницы. Например, для написанного нами шаблона:</p>
<p><code>&lt;h2&gt;&lt;$= this.header $&gt;&lt;/h2&gt;<br />
&lt;div class="content"&gt;&lt;$= this.content $&gt;&lt;/div&gt;</code></p>
<p>файл страницы должен содержать следующие поля:</p>
<p><code>$ header $<br />
Контактная информация<br />
<br />$ content $<br />
mail@example.com</code></p>
<p>Сгенерированная страница будет иметь вид:</p>
<p><code>&lt;h2&gt;Контактная информация&lt;/h2&gt;<br />
&lt;div class="content"&gt;mail@example.com&lt;/div&gt;</code></p>
<p>Никакие строки, кроме строк с&nbsp;названиями полей, не&nbsp;должны начинаться со&nbsp;знака <code>$</code>. Файл может содержать любое количество полей. Единственное обязательное поле&nbsp;&mdash; <code>header</code>, его содержание будет помещено в&nbsp;заголовок окна. Отсутствие этого поля приведет к&nbsp;ошибке при попытке открыть страницу.</p>
<p>Для корректной работы с&nbsp;кириллицей, сохраняйте все файлы в&nbsp;кодировке <nobr>UTF-8</nobr>.</p>

<h3>Добавление глобального содержания</h3>

<p>Глобальное содержание, то&nbsp;есть содержание, общее для всех страниц, хранится в&nbsp;файле <code>global.txt</code>. Отсутствие этого файла приводит к&nbsp;ошибке.</p>
<p>Формат файла <code>global.txt</code> такой&nbsp;же, как и&nbsp;файлов с&nbsp;содержанием страниц. Единственное различие заключается в&nbsp;том, что поля из&nbsp;<code>global.txt</code> доступны на&nbsp;каждой странице. Пример файла:</p>
<p><code>$ title $<br />
Мой сайт<br />
<br />$ author $<br />
Василий Пупкин</code></p>
<h3>Меню</h3>
<p>Меню&nbsp;&mdash; это перечисление ссылок на&nbsp;другие страницы в&nbsp;рамках одного блока. SNAC автоматически присвоит ссылке на&nbsp;текущую страницу класс <code><nobr>snac-current</nobr></code>. Чтобы эта функция заработала, в&nbsp;файле <code>global.txt</code> добавьте строки</p><p><code>$ menu $<br />
selector</code></p><p>где <code>selector</code>&nbsp;&mdash; это селектор блока, содержащего меню, в&nbsp;стиле jQuery. В&nbsp;целом синтаксис селекторов jQuery совпадает с&nbsp;селекторами CSS (<a href="http://api.jquery.com/category/selectors/">подробнее про&nbsp;селекторы jQuery</a>). Например, если меню находится в&nbsp;списке с&nbsp;классом <code>my_menu</code>, то запись должна быть следующей:</p>
<p><code>$ menu $<br />
ul.my_menu</code></p>

<h3>Ошибка 404</h3>

<p>Если посетитель сайта набирает адрес несуществующей страницы, SNAC перенаправляет его на&nbsp;страницу <code>#!404</code>. Вы&nbsp;можете настроить ее&nbsp;содержание редактированием файла <code>site/404.txt</code>.</p>

<h3>Создание бэкапа</h3>

<p>Для полного восстановления сайта достаточно иметь резервную копию директории <code>site/</code>. В&nbsp;ней хранится как содержание страниц, так и&nbsp;шаблон оформления. Директорию <code>system/</code> и&nbsp;файл <code>index.html</code> всегда можно взять из архива с&nbsp;дистрибутивом, который можно <a href="#!download">скачать</a> с&nbsp;этого сайта.</p>
